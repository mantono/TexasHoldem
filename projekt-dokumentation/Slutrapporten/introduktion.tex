\avsnitt{Introduktion}

Vi valde att skriva ett klassbibliotek till ett kortspel, eller mer specifikt till texashold'em poker. Vi använde oss utav Git som
versionshanteringsverktyg under hela projektets gång. Ungefär halvvägs in i projektet gjordes en
formell granskning av majoriteten av koden som vi hade skrivit för att senare utveckla det vi redan
hade skrivit till en bättre slutprodukt. För att kolla på statiska mått i vår kod använde vi Eclipse Metrics Plugin. Den profiler vi valde att använde var JVM Monitor i Eclipse.
För att uppnå högst möjliga coverage använde vi oss utav EclEmma för att säkerställa att vi uppnådde 100\% coverage på våra klasser, eller så nära det gick..
Under kodkritik körde vi igenom våra klasser med FindBugs för att se vad vi inte redan hade hittat som kunde ändras och möjligen göras bättre.

Källkod: \url{http://81.225.210.252/texas.tar.gz}

\begin{itemize}
	\item EclEmma - coverage
	\item Eclipse Metrics Plugin - software metrics
	\item GIT - versionshanteringssystem
	\item JUnit - enhetstestning
	\item FindBugs - kodkritik
	\item JVM Monitor - profiler
	\item Jflap - tillståndsmaskin
	\item ANT - byggscript
	\item Eclipse inbyggda debugger - debugger
\end{itemize}
